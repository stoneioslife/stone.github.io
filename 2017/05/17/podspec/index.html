PodSpec私有库

随着公司业务不断发展,由于业务直接的耦合比较严重,无法及时快速地实现各种个性化的需求。下定决心决心进行组件化改造，核心思路是效仿蘑菇街使用Router解耦各个组件，各个组件使用私有库的方式进行管理。Router这部分放在后面的文章进行总结,这篇文章主要总结在制作podspec时候遇到的坑.

##流程
1.第一步先在gitHub或者公司私有云上建立一个仓库，后面各个组件私有库都会push到这上面去，就好像gitHub的Master仓库一样。

2.第二步先将项目中Tools相关的类，单独创建了一个project,放到里面进行编译，解决问题。

3.Tools Project编译成功后,push到github或者公司私有库上面。然后创建podspec文件

4.打Tag.修改xxxx.podspecs文件

5.校验podspecs

6.返回修改之道validate success.然后上传到第一步命名的私有库上面

7.验证是否上传成功,是否能搜索到

问题汇总

1.normal i386 objective-c com.apple.compilers.llvm.clang.1_0.compiler
ToolsModule (0.0.12)

- ERROR | [ToolsModule/PayFramework] xcodebuild: Returned an unsuccessful exit code.

2.podspec中需要依赖自己定义的私有库

3.pod 命令 参数

4.需要引用pod管理的第三方的头文件.h

5.验证通过的podspec文件，进行打包

Macaca中文官网

环境配置(顺序无关)

请安装 Node.js v6.0 或者更高版本，装好 Node.js 后命令行里就已经集成了 npm 工具，为了提高安装模块的速度，请使用国内的 cnpm,我自己使用的是npm命令，速度也还可以.
以及Mac中 使用brew命令需要安装Homebrew（一款常用的 MacOS 的包管理器），请按照官网提示安装。

1.需要安装 usbmuxd 以便于通过 USB 通道测试 iOS 真机，不需要测试真机则不用安装

2.应用中如果包含Weview

3.安装 carthage

4.安装Macaca-cli Macaca-client Macaca-ios(驱动)

5.自己在摸索中还安装了devtool ios-deploy.目前还不知道具体作用

6.然后下载了ios-app-bootstrap-master,libimobiledevice-master,MacacaAutomation-master
